DevHealth


Microsoft DevHealth is a tool to dump the current memory map into a text-like file. There is also a viewer which runs on a Windows PC.
DevHealthViewer for OEM only.
DevHealth takes some time to dump the memory map into a file. So it is only useful for static analysis. The dump file lists all DLLs and processes with there memory usage and can be used to identify low memory conditions.

Installation

Copy DevHealth.exe onto a ARMv4i based Windows Mobile/CE device.

USAGE

Just start DevHealth.exe using mobile file explorer. Depending on the device taking the VM snap shot may take a long time (1-3 minutes). The screen will show the wait symbol until DevHealth has finished.

When finished you will find a new mem_x.txt on the first non-volatile volume (normally Flash File Store), where x is file counter which increments on every use.

OUTPUT

  • See sample\mem_1.txt. A list with all processes and there DLL use and linkage.
