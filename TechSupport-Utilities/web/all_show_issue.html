    The issue is intermittent - Go

Examples

  • Wrong device setup
    • Examples:
      • Barcode type to be read is not activated
      • WLAN settings do not match AP security settings
      • incompatible SDK runtime installed
      • trying to run a Windows Mobile application on a Windows CE device
  • incompatible wireless LAN security settings on Access Points
    • Examples:
      • WPA2-PSK with TKIP instead of AES on AP. This combination is not part of the standard
      • competitor WLAN controller has special setup for competitor devices
  • incorrect use of a SDK API function
    • Examples:
      • calling a function that awaits a different type than used
      • use an undocumented or private MS API function instead of official one
      • application is not aware of mobility restrictions
        • 32MB memory slots
        • power- awareness
        • wireless network connections can drop
      • Function keys have special function on Windows Mobile/Embedded Handheld!
  • incorrect use of HTML code
    • Examples:
      • using unsupported javascript or html code supported by Desktop Browsers
        • Hint: during development or for analysis enable ShowScriptErrors
      • using unsupported HTML5 code
        • not yet a standard
        • check browser by opening html5test.com and look at details to get a list of supported items
      • Function keys are not free to be used in Internet Explorer Mobile
  • Compared with other competitor device running without issue
    • Does the other device run the same OS?
      • Windows CE is not Windows Mobile nor Android
      • Windows Mobile 6.1 is not Windows Embedded Handheld 6.5 nor Windows Mobile 5
    • QVGA screen resolution is not the same as a WXGA or larger screen
