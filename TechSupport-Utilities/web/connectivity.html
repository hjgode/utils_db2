Connectivity

Possible issues with application using communication

Wireless

  • WLAN
  • WWAN
  • Bluetooth

Possible causes:

  • Non-tethered connections are prone to connection loss. Applications must aware of this and have a good connection recovery process.
  • For WLAN the wireless network setup might have an issue. Needs to be analyzed with a network trace.
  • Incorrect DNS or WINS setup.
  • WWAN connections may always change from connected to disconnected. Needs recovery process inside application code.
  • Bluetooth devices may get outside wireless range. Applications must have a recovery process.
  • Bluetooth devices may go to sleep and disconnect.

Possible helper tools for analysis

  • General network trace and analyzer tool like WireShark
  • MS Network Analyzer (MS Powertoys for Windows Mobile)
  • Bluetooth Analyzer
  • see Category Bluetooth
  • see Category WWAN

Wired

  • Ethernet
  • RS232
  • ActiveSync
  • USB

Possible causes:

  • Ethernet setup incorrect. For example duplicate IP addresses, wrong subnet setup.
  • Incorrect DNS or WINS setup.
  • For RS232 the connected devices must use the same setup for baudrate, stop bits, data size etc.
  • ActiveSync or the new Windows Mobile Device Center (WMDC) is used via RS232, USB or Bluetooth. WMDC can operate in PPP or RFNDIS mode. Possibly switching mode on device helps with USB setup.
  • For USB issues test always without any hub directly connected to PC mainboard. Use known good cables (some may be for charge only).
